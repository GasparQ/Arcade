25 /*
26  * Convert data from one encoding to another. Return:
27  *
28  * -2 : memory allocation failed
29  * -1 : unknown encoding
30  * 0 : data was converted exactly
31  * 1 : data was converted inexactly
32  * 2 : data was invalid (but still converted)
33  *
34  * We convert in two steps, via UTF-8, as this is the only
35  * reliable way of distinguishing between invalid input
36  * and valid input which iconv refuses to transliterate.
37  * We convert from UTF-8 twice, because we have no way of
38  * knowing whether the conversion was exact if iconv returns
39  * E2BIG (due to a bug in the specification of iconv).
40  * An alternative approach is to assume that the output of
41  * iconv is never more than 4 times as long as the input,
42  * but I prefer to avoid that assumption if possible.
43  */
